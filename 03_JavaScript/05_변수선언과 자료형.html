변수 선언 위치에 따른 구분

Java

        
            public class 클래스명{

                //필드
                private String name; // 인스턴스 변수 (클래스내 어디서든 사용 가능)
                public static int code; //static 변수, 클래스변수

                // 메소드
                public void method1(){
                    int num1 = 10; // method1의 지역 변수
                    
                    if(num1 == 10){
                        int num2 = 20; // if문의 지역 변수
                    }
                }
            }

JavaScript

        
            <script>
                
                // var == variable == 변수 선언 구문
                var 변수명; // 전역 변수 
                           // (같은 문서 내 선언된 이후부터 어디서든 사용 가능)
                
                변수명;    // 전역 변수로 선언됨
                          // (앞에 변수 선언 형식이 없어도 변수로 인식이됨)

                function 함수명(){

                    var 변수명; // function 지역 변수

                    변수명;     // 전역 변수

                    if(조건식){

                        var 변수명;   // function 지역변수
                        const 변수명; // if문 지역 변수
                        let 변수명;   // if문 지역 변수

                    }
                }
            </script>

var 변수 형식은 전역 변수 또는 function 지역변수만을 범위로 갖는다!!

변수 선언 형식


        var   : 변수 /  function level scope (함수 내부에서 어디서든 사용가능)
        const : 상수 / block {} level scope ( {}를 벗어나면 접근 불가 )
        let   : 변수 / block {} level scope ( {}를 벗어나면 접근 불가 )

        + var 변수는 변수명 중복이 가능하나,
          const,let은 변수명 중복이 불가능하다.

          1순위 : const
          2순위 : let (일반적인 변수를 사용할때)
          3순위 : var (es6 버전 이후부터는 잘 사용되지 않음)

자바스크립트의 자료형


        자바스크립트는 변수 선언시 별도의 자료형을 지정하지 않음.
        -> 변수에 대입되는 값(리터럴)에 의해서 변수의 자료형이 결정이 된다.

        - string(문자열)
            -> 자바스크립트는 문자 하나를 지정하는 char 자료형이 없다!
            -> 'a',"aaa" 모두 string
            -> '',"" 둘다 string 리터럴 기호 ==> 'hello', "a" 가능

        - number(숫자) -> 정수, 실수 모두 다

        - boolean(논리형) -> true / false

        - object(객체) -> Array, List, Set, Map
                        { name:"홍길동", age:15 } (자바스크립트 객체)
                        html 요소

        - function(함수) -> const testFn = function(){...} 
                                        // 이름이 없는 함수 == 익명함수

        - undefined(정의되지 않은 변수) == 변수를 선언만 하고 값 대입을 안함
            -> let str;
               console.log(str); // undefined
